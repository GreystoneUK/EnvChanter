# Contributing to EnvChanter

We appreciate your interest in contributing to EnvChanter! Here are a few guidelines:

## Getting Started

1. Fork the repository and clone your fork locally.
2. Create a new branch for your feature or bugfix:  
   `git checkout -b my-feature`
3. Make your changes and add tests where appropriate.

## Code Style

- This is a Go project. Please follow the [official Go style guidelines](https://golang.org/doc/effective_go.html).
- Format code using `gofmt` or `go fmt`.
- Keep pull requests focusedâ€”one feature or bugfix per PR.

## Commit Messages

- Write clear, descriptive commit messages.
- Reference related issues using `Fixes #issue_number` or `Closes #issue_number` if applicable.

## Testing

- Ensure all tests pass locally (`go test ./...`).
- Add new tests for new features and bugfixes.
- Submit your pull request only when all checks are passing.

## Pull Request Process

- Fill out the PR template.
- Keep PRs as small and focused as possible.
- Describe what your change is and why it is needed.
- One of the maintainers will review your PR, offer feedback, and merge when ready.

## Code of Conduct

All contributors are expected to follow our [Code of Conduct](CODE_OF_CONDUCT.md).

## Questions

For any questions, open an issue or start a discussion.

Thank you for helping make EnvChanter better!
